## Pas de « signes » d’IA dans l’historique public

Pour les **messages de commit**, **titres/description de PR**, **notes de release** et tout ce qui est **poussé sur le dépôt distant** :

- Ne pas ajouter de lignes du type **Co-authored-by:** … (outils / assistants).
- Ne pas mentionner d’outils d’assistance (**Cursor**, **Copilot**, noms de modèles, etc.) dans les commits ou les merges.
- Éviter les préfixes du type « généré par … », « assisté par … », ou emojis / badges qui identifient une IA.

Rédiger comme un contributeur humain : intention du changement, périmètre, risques éventuels — sans méta-commentaire sur la façon dont le texte a été produit.
